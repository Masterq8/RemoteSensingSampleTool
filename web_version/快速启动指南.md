# 快速启动指南

## 🚀 三种启动方式

### 方式一：一键启动（推荐新手）

**文件**: `启动全部服务.bat`

**操作步骤**:
1. 双击运行 `启动全部服务.bat`
2. 等待自动完成环境检查和服务启动
3. 打开浏览器访问 http://localhost:5173

**优点**:
- 自动检查环境
- 自动安装依赖
- 自动启动所有服务
- 显示详细的状态信息

**缺点**:
- 需要预先配置好环境

---

### 方式二：分别启动（推荐调试）

**文件**: `启动前端.bat` 和 `启动后端.bat`

**操作步骤**:
1. 打开第一个命令提示符窗口
   ```
   双击运行: 启动后端.bat
   ```
2. 打开第二个命令提示符窗口
   ```
   双击运行: 启动前端.bat
   ```
3. 打开浏览器访问 http://localhost:5173

**优点**:
- 可以分别查看前后端日志
- 便于调试问题
- 可以单独重启某个服务

**缺点**:
- 需要手动打开两个窗口

---

### 方式三：手动命令行启动（推荐高级用户）

#### 步骤 1: 启动后端

打开命令提示符（CMD），执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version
python main.py
```

等待看到以下输出表示启动成功：
```
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

#### 步骤 2: 启动前端

打开**新的**命令提示符窗口，执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version\vue-app
npm install
npm run dev
```

等待看到以下输出表示启动成功：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
```

#### 步骤 3: 访问应用

打开浏览器访问: http://localhost:5173

**优点**:
- 完全控制启动过程
- 可以看到完整的错误信息
- 便于自定义参数

**缺点**:
- 需要手动输入命令
- 需要了解命令行操作

---

## 🛠️ 辅助工具

### 环境检查

**文件**: `环境检查.bat`

**用途**: 检查所有必需的环境和依赖是否已安装

**运行时机**:
- 首次使用前
- 遇到启动问题时
- 更新系统后

**检查项目**:
- Python 环境
- ArcPy 模块
- Node.js 环境
- npm 工具
- 端口占用情况
- 前端依赖
- 文件权限

---

### 服务测试

**文件**: `服务测试.bat`

**用途**: 测试服务是否正常运行

**运行时机**:
- 服务启动后
- 遇到连接问题时
- 验证修复效果

**测试项目**:
- 后端服务响应
- 前端服务响应
- API 端点响应

---

### 停止服务

**文件**: `停止全部服务.bat`

**用途**: 停止所有运行中的服务

**运行时机**:
- 不再使用服务时
- 需要重启服务时
- 端口被占用时

**停止项目**:
- 后端服务窗口
- 前端服务窗口
- 残留的 Python 进程
- 残留的 Node.js 进程

---

## 📋 启动前检查清单

在启动服务之前，请确认以下项目：

- [ ] 已安装 Python 2.7（推荐使用 ArcGIS 自带版本）
- [ ] 已安装 ArcGIS Desktop 或 ArcGIS Pro
- [ ] 已安装 Node.js 18 或更高版本
- [ ] 端口 8000 未被占用
- [ ] 端口 5173 未被占用
- [ ] 已安装前端依赖（首次运行会自动安装）
- [ ] 有有效的 ArcGIS 授权许可

---

## 🔍 故障排除

如果服务启动失败，请按以下顺序操作：

1. **运行环境检查**
   ```
   双击运行: 环境检查.bat
   ```

2. **查看错误日志**
   - 后端错误会显示在后端服务窗口
   - 前端错误会显示在前端服务窗口

3. **参考故障排除指南**
   ```
   打开文件: 故障排除指南.md
   ```

4. **尝试手动启动**
   - 使用方式三手动启动，查看详细错误信息

---

## 📞 获取帮助

如果以上方法都无法解决问题：

1. 收集以下信息：
   - 错误截图
   - 环境检查输出
   - 服务日志
   - 系统信息

2. 查看 `故障排除指南.md` 获取详细解决方案
