# 服务启动故障排除指南

## 快速诊断

如果服务启动失败，请按以下顺序检查：

1. **运行环境检查**
   ```
   双击运行: 环境检查.bat
   ```
   这会自动检测所有必需的环境和依赖。

2. **查看错误日志**
   - 后端错误会显示在后端服务窗口
   - 前端错误会显示在前端服务窗口

## 手动启动服务（替代方案）

### 方案一：分别启动服务（推荐用于调试）

#### 步骤 1: 启动后端服务

打开命令提示符（CMD），执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version
python main.py
```

**预期输出：**
```
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

**验证后端服务：**
- 打开浏览器访问: http://localhost:8000/docs
- 应该看到 FastAPI 的 API 文档页面

#### 步骤 2: 启动前端服务

打开**新的**命令提示符窗口，执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version\vue-app
npm install
npm run dev
```

**预期输出：**
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

**验证前端服务：**
- 打开浏览器访问: http://localhost:5173
- 应该看到遥感样本制作工具的界面

### 方案二：使用 PowerShell 启动（高级用户）

#### 创建启动脚本

创建文件 `start-services.ps1`：

```powershell
# 启动后端服务
$backendJob = Start-Job -ScriptBlock {
    cd "f:\RemoteSensingSampleTool\web_version"
    python main.py
}

# 等待后端启动
Start-Sleep -Seconds 3

# 启动前端服务
$frontendJob = Start-Job -ScriptBlock {
    cd "f:\RemoteSensingSampleTool\web_version\vue-app"
    npm run dev
}

Write-Host "后端服务: http://localhost:8000"
Write-Host "前端服务: http://localhost:5173"
Write-Host "API文档: http://localhost:8000/docs"
Write-Host ""
Write-Host "按 Ctrl+C 停止所有服务"

# 等待用户中断
try {
    while ($true) {
        Start-Sleep -Seconds 1
    }
} finally {
    Stop-Job $backendJob
    Stop-Job $frontendJob
    Remove-Job $backendJob
    Remove-Job $frontendJob
}
```

运行脚本：
```powershell
powershell -ExecutionPolicy Bypass -File start-services.ps1
```

## 常见问题及解决方案

### 问题 1: "未找到 Python" 错误

**症状：**
```
'python' 不是内部或外部命令，也不是可运行的程序
```

**解决方案：**

1. **确认 Python 安装路径**
   - ArcGIS Python 通常在: `C:\Python27\ArcGIS10.4\python.exe`
   - 或: `C:\Python27\ArcGIS10.2\python.exe`

2. **临时解决方案（修改批处理文件）**
   ```cmd
   REM 将 "python main.py" 改为完整路径
   C:\Python27\ArcGIS10.4\python.exe main.py
   ```

3. **永久解决方案（添加到 PATH）**
   - 右键"此电脑" → 属性 → 高级系统设置
   - 环境变量 → 系统变量 → Path → 编辑
   - 添加: `C:\Python27\ArcGIS10.4\`
   - 重启命令提示符

### 问题 2: "未找到 ArcPy" 错误

**症状：**
```
ImportError: No module named arcpy
```

**解决方案：**

1. **确认 ArcGIS 安装**
   - ArcPy 只随 ArcGIS Desktop 或 ArcGIS Pro 安装
   - 单独安装 Python 不会包含 ArcPy

2. **使用正确的 Python**
   - 必须使用 ArcGIS 自带的 Python 环境
   - 不要使用系统安装的 Python 3.x

3. **验证 ArcPy**
   ```cmd
   C:\Python27\ArcGIS10.4\python.exe -c "import arcpy; print(arcpy.GetInstallInfo()['Version'])"
   ```

### 问题 3: "未找到 Node.js" 错误

**症状：**
```
'node' 不是内部或外部命令
```

**解决方案：**

1. **安装 Node.js**
   - 访问: https://nodejs.org/
   - 下载 LTS 版本（推荐 18.x 或 20.x）
   - 安装时确保勾选 "Add to PATH"

2. **验证安装**
   ```cmd
   node --version
   npm --version
   ```

3. **重新安装前端依赖**
   ```cmd
   cd f:\RemoteSensingSampleTool\web_version\vue-app
   rmdir /s /q node_modules
   npm install
   ```

### 问题 4: 端口已被占用

**症状：**
```
OSError: [WinError 10048] 通常每个套接字地址
```

**解决方案：**

1. **查找占用端口的进程**
   ```cmd
   netstat -ano | findstr ":8000"
   netstat -ano | findstr ":5173"
   ```

2. **停止占用端口的进程**
   ```cmd
   taskkill /PID <进程ID> /F
   ```

3. **或使用停止服务脚本**
   ```
   双击运行: 停止全部服务.bat
   ```

### 问题 5: 前端依赖安装失败

**症状：**
```
npm ERR! code ENOENT
npm ERR! syscall open
```

**解决方案：**

1. **清理缓存**
   ```cmd
   npm cache clean --force
   ```

2. **删除 node_modules 重新安装**
   ```cmd
   cd f:\RemoteSensingSampleTool\web_version\vue-app
   rmdir /s /q node_modules
   npm install
   ```

3. **使用淘宝镜像（如果网络问题）**
   ```cmd
   npm config set registry https://registry.npmmirror.com
   npm install
   ```

### 问题 6: 后端启动失败 - ArcGIS 授权问题

**症状：**
```
RuntimeError: NotInitialized
```

**解决方案：**

1. **确认 ArcGIS 授权**
   - 打开 ArcMap 或 ArcCatalog
   - 检查是否有有效的授权许可

2. **初始化 ArcGIS 授权**
   ```cmd
   C:\Python27\ArcGIS10.4\python.exe -c "import arcpy; arcpy.CheckOutExtension('Spatial')"
   ```

3. **检查授权文件**
   - ArcGIS 授权文件通常在: `C:\Program Files (x86)\ArcGIS\License10.x\`

### 问题 7: CORS 跨域错误

**症状：**
```
Access to XMLHttpRequest has been blocked by CORS policy
```

**解决方案：**

1. **检查后端 CORS 配置**
   - 确认 `main.py` 中已启用 CORS
   - 检查允许的源地址配置

2. **临时解决方案（仅开发环境）**
   - 使用浏览器插件禁用 CORS（不推荐生产环境）

### 问题 8: 文件上传失败

**症状：**
```
413 Request Entity Too Large
```

**解决方案：**

1. **检查文件大小限制**
   - 查看 `main.py` 中的上传大小配置
   - 调整 `max_upload_size` 参数

2. **检查磁盘空间**
   - 确保输出目录有足够的磁盘空间

## 服务状态验证

### 验证后端服务

```cmd
curl http://localhost:8000/docs
```

或打开浏览器访问: http://localhost:8000/docs

### 验证前端服务

```cmd
curl http://localhost:5173
```

或打开浏览器访问: http://localhost:5173

### 检查进程状态

```cmd
tasklist | findstr "python.exe"
tasklist | findstr "node.exe"
```

## 日志查看

### 后端日志
- 运行时日志显示在后端服务窗口
- 可以重定向到文件:
  ```cmd
  python main.py > backend.log 2>&1
  ```

### 前端日志
- 运行时日志显示在前端服务窗口
- 浏览器控制台: 按 F12 打开开发者工具

## 性能优化建议

1. **后端优化**
   - 使用生产级服务器（如 Gunicorn + Uvicorn）
   - 启用缓存机制
   - 优化数据库查询

2. **前端优化**
   - 构建生产版本: `npm run build`
   - 使用 CDN 加速
   - 启用 gzip 压缩
