# 服务启动脚本使用说明

## 📁 可用的启动脚本

### 1. 批处理文件 (.bat)

| 文件名 | 说明 | 推荐场景 |
|--------|------|----------|
| `启动全部服务.bat` | 一键启动所有服务 | 日常使用 |
| `启动前端.bat` | 仅启动前端服务 | 调试前端 |
| `启动后端.bat` | 仅启动后端服务 | 调试后端 |
| `停止全部服务.bat` | 停止所有服务 | 停止服务 |
| `环境检查.bat` | 检查环境配置 | 故障排查 |
| `服务测试.bat` | 测试服务状态 | 验证服务 |

### 2. PowerShell 脚本 (.ps1)

| 文件名 | 说明 | 推荐场景 |
|--------|------|----------|
| `启动全部服务.ps1` | 一键启动所有服务（推荐） | 日常使用 |
| `停止全部服务.ps1` | 停止所有服务（推荐） | 停止服务 |

---

## 🚀 快速开始

### 方法一：使用 PowerShell 脚本（推荐）

#### 启动服务
```powershell
# 右键点击 "启动全部服务.ps1" → "使用 PowerShell 运行"
# 或在 PowerShell 中执行：
.\启动全部服务.ps1
```

#### 停止服务
```powershell
# 右键点击 "停止全部服务.ps1" → "使用 PowerShell 运行"
# 或在 PowerShell 中执行：
.\停止全部服务.ps1
```

**优点**：
- 更好的中文支持
- 彩色输出，易于阅读
- 自动打开浏览器
- 更详细的错误提示

---

#### 停止服务
```
双击运行: 停止全部服务.bat
```

### 方法三：手动启动（调试用）

#### 步骤 1: 启动后端

打开命令提示符（CMD），执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version
python main.py
```

#### 步骤 2: 启动前端

打开**新的**命令提示符窗口，执行：

```cmd
cd f:\RemoteSensingSampleTool\web_version\vue-app
npm install
npm run dev
```

#### 步骤 3: 访问应用

打开浏览器访问: http://localhost:5173

## 📋 启动前检查清单

在启动服务之前，请确认：

- [ ] 已安装 Python 2.7（推荐使用 ArcGIS 自带版本）
- [ ] 已安装 ArcGIS Desktop 或 ArcGIS Pro
- [ ] 已安装 Node.js 18 或更高版本
- [ ] 端口 8000 未被占用
- [ ] 端口 5173 未被占用
- [ ] 已安装前端依赖（首次运行会自动安装）
- [ ] 有有效的 ArcGIS 授权许可

## 🔍 故障排除

### 问题：无法启动服务

1. **运行环境检查**
   ```
   双击运行: 环境检查.bat
   ```

2. **查看错误日志**
   - 后端错误会显示在后端服务窗口
   - 前端错误会显示在前端服务窗口

3. **参考故障排除指南**
   ```
   打开文件: 故障排除指南.md
   ```

### 问题：端口被占用

1. **停止所有服务**
   ```
   双击运行: 停止全部服务.ps1
   ```

2. **检查端口占用**
   ```cmd
   netstat -ano | findstr ":8000"
   netstat -ano | findstr ":5173"
   ```

3. **手动停止占用端口的进程**
   ```cmd
   taskkill /PID <进程ID> /F
   ```

### 问题：依赖安装失败

1. **清理缓存**
   ```cmd
   npm cache clean --force
   ```

2. **重新安装依赖**
   ```cmd
   cd f:\RemoteSensingSampleTool\web_version\vue-app
   rmdir /s /q node_modules
   npm install
   ```

3. **使用淘宝镜像（如果网络问题）**
   ```cmd
   npm config set registry https://registry.npmmirror.com
   npm install
   ```
